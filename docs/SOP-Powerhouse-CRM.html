1. Getting Started

What Is This?

2. Core Concepts

Assets

An asset is any real-world item PleoChrome is working to monetize. Each asset has a name, asset type (gemstone, real estate, precious metal, mineral rights, or other), a value model (how you plan to monetize it), and an auto-generated reference code.

The Six Phases

Every asset moves through six phases in order:

#PhaseWhat Happens
1LeadInitial sourcing and identification
2IntakeKYC/KYB verification, holder screening, preliminary assessment
3MaturityFull appraisal, custody arrangement, insurance
4SecurityLegal structuring (SPV, securities filings, compliance)
5Value CreationToken/share creation, smart contracts, offering setup
6DistributionMarketing, investor onboarding, trading/settlement

Stages, Tasks, and Subtasks

Within each phase:

  • Stages are major milestones (e.g., "Holder Qualification & Screening")
  • Tasks are specific work items within a stage (e.g., "Run OFAC/SDN screening")
  • Subtasks break tasks into smaller checkable items

All stage names, task titles, and subtask titles can be renamed at any time to match your specific situation.

3. Daily Workflow

Creating a New Asset

  1. Click + New Asset from the Pipeline page
  2. Fill in the wizard:
    • Step 1: Name, asset type, holder entity
    • Step 2: Value model selection
    • Step 3: Type-specific details (carat weight, property address, etc.)
    • Step 4: Review and create
  3. The system automatically applies the appropriate workflow template

Working Through the Workflow

  1. Open an asset from the Pipeline or Assets list
  2. You land on the Workflow tab — this is where you do your work
  3. Tap/click a stage to see its tasks
  4. For each task:
    • Open to see full details, subtasks, files, comments
    • Mark subtasks as complete with the checkbox
    • Add notes, attach files, leave comments
    • Change status: To Do → Active → Done
  5. When all tasks in a stage are done, mark the stage as complete
  6. When a phase is complete, advance to the next phase
On Mobile: The workflow uses a flat drill-down — tap a stage to enter it, tap a task to see details. Use the back arrow to navigate up. Tap the pencil icon to rename any stage or task.

Advancing Phases

Click Advance to [Phase] (desktop) or find it in the ... menu (mobile). The system warns if tasks are incomplete — you can override with a reason that gets logged for audit.

6. Templates

How Templates Work

When you create an asset and select a value model, the system applies two templates:

  1. Shared Pipeline — phases 1–3, universal for all value models
  2. Value Model template — phases 4–6, specific to your chosen monetization path

Customizing After Application

Once applied, all stages, tasks, and subtasks are independent copies. You can freely:

  • Rename any stage, task, or subtask
  • Add new stages, tasks, or subtasks
  • Delete irrelevant items
  • Reorder items (desktop: drag-and-drop)

Changes do NOT affect the template or other assets.
